Contact

Dontprint was developed and is maintained by Robert Bamler. You are welcome to send me feedback but please read through the list of frequently asked questions first.

Getting help

  1. Before asking any questions please read through the list of frequently asked questions.
  2. Only if your question is not already answered in the list of frequently asked questions, you are welcome to report a bug or contact me personally.

Getting involved

  • If you found a bug in Dontprint please file a bug report. Please note that if your bug is about layouting problems then you should probably report it to the author of k2pdfopt rather than here.
  • If you have an idea for a new feature please create a new issue and add the label "enhancement". Criticism is also very much welcome via the same channel but please be constructive and to the point.
  • If you would like to contribute some code or artwork the easiest way is to fork the repository on GitHub and then start a pull request. Feel free to contact me if you experience any problems.

Redistributing and code reusing

You may redistribute Dontprint and/or reuse the source code of Dontprint in your own code as long as you comply with the license agreement. I appreciate a short e-mail notice if you use parts of Dontprint in your code.
